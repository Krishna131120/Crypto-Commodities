================================================================================
COMPLETE .env FILE TEMPLATE (Alpaca + Angel One)
================================================================================

Copy this to your .env file and fill in your actual credentials:

================================================================================
# ALPACA API (for Crypto Trading)
================================================================================
# Get these from: https://app.alpaca.markets/paper/dashboard/overview
ALPACA_API_KEY="your_alpaca_api_key_here"
ALPACA_SECRET_KEY="your_alpaca_secret_key_here"
ALPACA_BASE_URL="https://paper-api.alpaca.markets"

# Alternative Alpaca variable names (if your code uses these):
# APCA_API_KEY_ID="your_alpaca_api_key_here"
# APCA_API_SECRET_KEY="your_alpaca_secret_key_here"
# APCA_API_BASE_URL="https://paper-api.alpaca.markets"

================================================================================
# ANGEL ONE API (for MCX Commodities Trading)
================================================================================
# Register for API key at: https://smartapi.angelone.in/
# Enable TOTP in your Angel One account
ANGEL_ONE_API_KEY="your_angel_one_api_key_here"
ANGEL_ONE_CLIENT_ID="your_client_id"
ANGEL_ONE_PASSWORD="your_trading_password_or_mpin"
ANGEL_ONE_TOTP_SECRET="your_totp_secret"  # Optional if using pyotp library

# Optional: Override Angel One base URL (default: https://apiconnect.angelbroking.com)
# ANGEL_ONE_BASE_URL="https://apiconnect.angelbroking.com"

================================================================================
IMPORTANT NOTES
================================================================================

1. TOTP Setup:
   - Enable TOTP in your Angel One account
   - Scan QR code with authenticator app (Google Authenticator, etc.)
   - Use TOTP secret for automatic token generation

2. Authentication:
   - Login requires: Client ID + Password + TOTP
   - Access token valid for session duration
   - System will auto-refresh token when expired

3. Security:
   - .env file is in .gitignore (won't be committed to git)
   - Never share your .env file
   - Keep your tokens secret
